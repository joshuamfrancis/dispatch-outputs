# GitHub Paid Plans: Compare and Contrast

*Source: [github.com/pricing](https://github.com/pricing), as of July 2026*

GitHub offers three core plans — Free, Team, and Enterprise — plus a set of usage-based add-ons (Copilot, Advanced Security, Codespaces, LFS). Below is a comparison of the two **paid** tiers, Team and Enterprise, with Free included as a baseline.

## At a Glance

| | Free | Team | Enterprise |
|---|---|---|---|
| Price | $0/month | $4/user/month (first 12 months) | $21/user/month (first 12 months), starting price |
| Target audience | Individuals, small open source projects | Growing teams and organizations | Large orgs with compliance/security needs |
| Private collaborators | Unlimited | $4/user/month | $21/user/month |
| Actions minutes/month | 2,000 | 3,000 | 50,000 |
| Packages storage | 500 MB | 2 GB | 50 GB |
| Codespaces | $0 spend limit, with policies | Ability to increase spend limit | Ability to increase spend limit |

All three plans include free minutes/storage for public repositories, unlimited public and private repos, unlimited GitHub Apps, and unlimited collaborators on public repos.

## Team Plan — $4/user/month

Team is built for organizations that need real collaboration controls beyond what Free offers. Everything in Free is included, plus:

- Repository rules (branch/tag protection, required reviewers)
- Multiple reviewers and code owners on pull requests
- Draft pull requests
- Pages and Wikis (private repos)
- Environment deployment branches and secrets
- Web-based support (vs. community forum only on Free)
- Access to GitHub Codespaces with configurable spend limits
- Featured add-ons available: GitHub Secret Protection, GitHub Code Security

This tier suits small-to-midsize teams that need branch protection, reviewer workflows, and basic organizational controls, without needing identity federation or compliance certifications.

## Enterprise Plan — starting at $21/user/month

Enterprise includes everything in Team, plus features aimed at large organizations, regulated industries, and compliance-heavy environments:

- SAML single sign-on (SSO) and Enterprise Managed Users
- SCIM user provisioning tied to your identity provider
- Enterprise Account for centrally managing multiple organizations
- Data residency (regional cloud deployment on Microsoft Azure)
- Advanced audit log and Audit Log API
- SOC 1 / SOC 2 Type 2 reports annually
- FedRAMP Tailored ATO (for U.S. government use cases)
- Environment protection rules and enterprise-wide repository rules
- GitHub Connect (bridges Enterprise Server and Enterprise Cloud)
- 50,000 Actions minutes/month and 50 GB Packages storage
- Exclusive add-on: Premium and Premium Plus Support (30-minute SLA on urgent tickets, 24/7 phone support, optional dedicated Customer Reliability Engineer)

Enterprise is the only tier offering self-hosted deployment (GitHub Enterprise Server) for organizations that need on-prem or self-managed cloud hosting.

## Add-Ons (available on top of any plan)

These are billed separately and are not included in the base plan prices above:

- **GitHub Copilot** — free tier with limited completions, paid tiers scale up for individuals and Copilot for Business/Enterprise
- **GitHub Advanced Security** — code scanning, secret scanning, and dependency review at scale
- **GitHub Codespaces** — usage-based, from $0.18/hour compute and $0.07/GB storage
- **Git Large File Storage (LFS)** — $5/month for 50 GB bandwidth and 50 GB storage
- **GitHub Models** — AI model access for workflows

## Bottom Line

Team is the entry point for any organization that needs branch protection and reviewer workflows, at a low per-user cost. Enterprise is priced for organizations that specifically need SSO/SCIM, audit logging, compliance reporting (SOC 2, FedRAMP), data residency, or self-hosted deployment — features aimed at governance and scale rather than day-to-day developer workflow. Smaller teams without compliance requirements will typically find Team sufficient; the jump to Enterprise is driven by IT/security policy requirements more than by feature density.
